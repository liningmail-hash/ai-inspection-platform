# 总体方案与项目流程

## 项目定位

AI巡检平台一期面向园区/工厂小型试点，目标是用一套轻量但可扩展的私有云边协同架构，跑通摄像头 AI 巡检、无人机巡检、告警处置、样本标注、模型训练和模型发布闭环。

一期不是做大而全的平台，不追求一次性覆盖所有厂家 SDK 和所有算法，而是先建立稳定的架构骨架、统一接口、核心流程和工程纪律。

## 目标规模

- 摄像头：50-150 路接入。
- AI 并发：10-20 路视频 AI 分析。
- 无人机：1 套机场/无人机。
- 部署：单园区私有化部署，中心服务 + 边缘节点。
- 训练：单机 GPU 起步，后续扩展 GPU 队列。

## 业务闭环

1. 接入设备：摄像头、通道、边缘节点、无人机机场、无人机。
2. 配置巡检：选择点位、算法、ROI、时段、阈值、告警等级。
3. 下发任务：中心平台把巡检计划下发到边缘节点。
4. 边缘分析：边缘节点拉流、转码、推理、生成证据。
5. 告警处置：中心平台接收告警，推送前端，完成确认、误报、派单、关闭。
6. 样本回流：告警证据和误报/漏报样本进入数据集。
7. 标注训练：算法工程师标注、质检、启动训练、查看指标。
8. 模型发布：模型候选版本灰度到边缘节点，验证后进入生产，必要时回滚。
9. 无人机巡检：机场执行固定航线，视频流进入 AI 分析，生成巡检记录和报告。

## 系统组件

| 组件 | 职责 | 当前实现 |
|---|---|---|
| 前端工作台 | 业务操作入口、状态可视化、告警处置 | Vue 工程骨架 |
| 中心后端 | 业务主数据、权限、巡检计划、告警、模型发布 | Spring Boot API 骨架 |
| AI 服务 | 数据集、训练任务、模型版本、推理接口 | FastAPI 骨架 |
| 边缘节点 | 设备接入、视频处理、推理任务、事件缓存 | FastAPI 模拟器 |
| PostgreSQL | 结构化业务数据 | 初始化脚本 |
| Redis | 状态缓存、任务队列预留 | Compose 预留 |
| MinIO | 证据图片、短视频、训练样本、模型文件 | Compose 预留 |
| MQTT | 中心与边缘消息通道 | 架构预留 |

## 开发阶段

### Phase 1：工程骨架

- 建立前端、后端、AI服务、边缘模拟器、数据库、部署目录。
- 完成前端 10 个核心页面入口。
- 完成中心后端第一批 REST API。
- 完成 AI 服务和边缘模拟器接口骨架。
- 完成开源基础材料和工程规范。

### Phase 2：核心联调

- 前端服务层从 Mock 数据切换到后端 API。
- 后端建立 Repository/QueryService 分层，再接入 PostgreSQL，替换内存演示数据。
- 告警状态流转、训练任务、模型发布形成端到端链路。
- 边缘模拟器定时上报设备状态和告警事件。

### Phase 3：真实能力接入

- 接入真实视频播放链路。
- 验证 RTSP、ONVIF、GB/T 28181、JT/T 1078。
- 接入首个无人机机场厂家。
- 接入真实 AI 推理模型。

### Phase 4：试点验收

- 100路视频在线验证。
- 10-20路并发 AI 验证。
- 72小时稳定性测试。
- 完成部署手册、验收报告、演示数据脱敏。

## 项目管理原则

- 先闭环，再扩展。
- 先接口稳定，再接真实厂家。
- 先模拟可跑，再替换真实实现。
- 每个模块必须有清晰边界和负责人视角。
- 新增功能必须能解释它属于哪个业务闭环，不能只因为“可能有用”就加入。
