# 真实业务功能开发路线图

更新时间：2026-06-16

本文档由总指挥维护，用于把项目从“演示 Mock 环境”推进到“可真实使用的生产安全园区 AI 巡检系统”。

## 1. 当前结论

当前云端 Mock 环境已经可以用于联调和内部演示，但它不是最终目标。

下一阶段正式进入 P0-REAL：真实业务可操作闭环开发。

核心要求：

- 页面按钮必须能产生真实业务动作。
- 数据必须真实保存到后端和数据库。
- 后端必须返回明确成功或失败结果。
- 关键操作必须进入审计日志。
- Mock 只能作为无真实设备时的兜底，不能替代业务闭环。

## 2. P0-REAL 总目标

先做一个可真实操作的最小闭环：

1. 用户登录进入系统。
2. 管理员新增一个厂区摄像头、无人机或车载终端。
3. 系统保存设备和通道。
4. 用户测试连接或同步通道。
5. 用户在视频墙创建播放会话。
6. 用户给通道创建 AI 任务。
7. 边缘或 AI 服务上报告警事件。
8. 告警中心展示证据并允许确认、误报、派单、关闭。
9. 全过程形成审计日志。

## 3. 第一轮任务分派

### BE-REAL-P0-001：后端设备管理真实 CRUD

负责人：程序员 A

目标：

- 让设备管理不再只是静态展示。
- 支持厂区摄像头、无人机、车载终端的新增、编辑、删除、查询、测试连接、同步通道。

范围：

- 允许修改 `backend/`、`database/`、`docs/API_CONTRACTS.md`。
- 不修改前端页面实现。

必须实现：

- `GET /api/devices` 查询设备列表。
- `POST /api/devices` 新增设备。
- `PUT /api/devices/{id}` 编辑设备。
- `DELETE /api/devices/{id}` 删除设备或软删除。
- `POST /api/devices/{id}/test` 测试连接。
- `POST /api/devices/{id}/sync-channels` 同步视频通道。
- 设备字段至少包含：名称、sourceType、vendor、endpoint、credentialRef、location、edgeNodeId、status、createdAt、updatedAt。
- 新增、编辑、删除、测试、同步必须写审计日志。
- 敏感凭据不存明文，只存 `credentialRef`。

验收口径：

- 新增设备后刷新页面或重新请求接口仍能查到。
- 编辑设备后字段真实变化。
- 删除后列表不再显示或显示为 disabled。
- 测试连接能返回 success/failed 和明确 message。
- 同步通道能产生或更新 video channel。

返回总指挥格式：

```text
任务编号：BE-REAL-P0-001
当前状态：
新增/修改接口：
数据库变化：
审计日志覆盖项：
自测结果：
是否影响现有云端 Mock：
需要前端 B 对接的字段：
发现的问题：
```

### FE-REAL-P0-001：前端设备管理真实表单

负责人：程序员 B

目标：

- 让设备管理页面从“只看”变成“能新增、编辑、删除、测试、同步”。

范围：

- 允许修改 `frontend/`。
- API 入口仍统一走 `frontend/src/services/api.ts`。
- 不修改后端、数据库、部署文件。

必须实现：

- “添加设备”弹窗或页面。
- “编辑设备”弹窗或页面。
- 删除确认。
- 测试连接按钮反馈。
- 同步通道按钮反馈。
- sourceType 选择：camera、drone、vehicle。
- vendor 选择：MOCK_VENDOR、HIKVISION、DAHUA、DJI_DOCK、JT1078。
- endpoint、credentialRef、location、edgeNodeId 等字段录入。
- 后端不可用时可以显示 Mock 兜底，但必须明确标识，不得伪装成真实保存成功。

验收口径：

- 用户能完成新增设备表单填写。
- 提交失败时页面显示失败原因。
- 提交成功后列表刷新。
- 编辑、删除、测试、同步均有明确状态反馈。

返回总指挥格式：

```text
任务编号：FE-REAL-P0-001
当前状态：
新增/修改页面：
新增/修改 API 方法：
表单字段：
自测结果：
需要后端 A 提供或确认的字段：
发现的问题：
```

## 4. 第一轮不做什么

为了避免任务失控，第一轮暂不做：

- 多园区。
- Kubernetes。
- 三维地图。
- 大模型视觉分析。
- 复杂模型训练流水线。
- 真实无人机远程控制。
- 真实 50-150 路并发压测。

这些放到 P1/P2，等设备管理、视频、AI、告警闭环先真实可用后再推进。

## 5. 总指挥验收标准

第一轮完成后，必须能演示：

```text
新增设备 -> 保存成功 -> 同步通道 -> 视频墙选择通道 -> 创建播放会话 -> 创建 AI 任务 -> 上报告警 -> 告警处置 -> 审计可查
```

如果某个环节只能展示假数据，不算通过。
