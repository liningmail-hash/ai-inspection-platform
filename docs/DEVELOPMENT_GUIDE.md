# 开发规范

这份文档用于防止项目在迭代中变成难维护的代码堆。所有新增功能都应遵守这里的工程约束。

团队分工、任务单、回报格式和冲突处理统一遵循 [团队协同与总指挥执行手册](TEAM_COMMAND_CENTER.md)。

## 总体原则

- 小步提交，功能闭环优先。
- 不跨层偷懒：前端不写业务真规则，后端不混入厂商 SDK 细节，AI 服务不承担平台权限逻辑。
- 接口先定契约，再写实现。
- 模拟实现必须标明边界，后续能替换真实实现。
- 不为了赶进度复制粘贴大段相似代码。

## 目录边界

### frontend

- `views/`：页面级组件，只组合业务区域，不堆复杂业务逻辑。
- `components/`：可复用 UI 组件。
- `services/`：API 请求和数据适配。
- `data/`：仅存演示/Mock 数据，不能长期承载真实业务状态。
- `style.css`：当前阶段可作为全局样式入口，组件规模扩大后拆分为主题、布局、组件样式。

### backend

- `api/`：Controller，只处理 HTTP 入参、出参和状态码。
- `service/`：业务流程编排。
- `repository/`：数据访问抽象和实现。
- `domain/`：核心业务对象和领域概念。
- `adapter/`：设备、无人机、外部系统适配接口。
- 后续新增 `repository/` 承载数据库访问，不允许 Controller 直接查数据库。

### ai-service

- 只负责 AI 相关能力：推理、训练、数据集、模型指标、模型文件。
- 不处理用户权限、站点权限、告警流转审批。
- 模型框架实现必须隔离，避免把 PyTorch/ONNX/TensorRT 代码散落在路由函数里。

### edge-simulator

- 当前是模拟器，不代表真实边缘 Agent 的最终实现。
- 负责模拟设备状态、推理任务、事件上报、断网缓存。
- 后续真实边缘节点应保持相同接口，不影响中心平台。

## 代码质量要求

- 单个函数尽量不超过 60 行，超过时优先拆成有业务含义的小函数。
- 单个文件尽量不超过 300 行，超过时按职责拆分。
- 命名必须表达业务含义，避免 `data1`、`temp`、`handleThing`。
- 不允许把大量 if/else 写进页面组件或 Controller。
- 不允许在业务代码里硬编码真实账号、密码、IP、Token、RTSP 地址。
- 不允许把厂商 SDK 调用直接散落在业务服务里，必须经过适配器。
- 不允许吞掉异常。失败要能被日志、接口响应或任务状态看见。
- 不允许为了 UI 展示伪造真实生产数据，演示数据要明确标识。

## 前端规范

- 页面风格遵循 `docs/UI_GUIDELINES.md`。
- 优先用组件组合，不在一个 `.vue` 文件里堆完整系统。
- 状态要有文字和颜色，不能只靠颜色表达。
- 表格、筛选、详情抽屉、状态标签要保持统一交互。
- 接口调用集中在 `services/`，页面不直接拼 URL。
- Mock 数据要能被 API 返回结构自然替换。

## 后端规范

- REST API 使用 `/api/...` 前缀。
- 返回对象使用明确字段，不返回无结构字符串。
- 状态值使用稳定英文枚举，前端再做中文展示。
- 设备、无人机、AI 模型都通过抽象接口扩展。
- 告警、模型发布、训练任务必须保留审计字段。
- 后续接数据库时必须引入迁移工具，建议 Flyway。
- Controller 不允许直接依赖具体数据实现，只能依赖 Service。
- Service 不允许直接写 SQL，数据库访问必须通过 Repository。
- 临时演示数据必须放在 InMemory Repository，不能散落在 Controller 或页面里。

## Python 服务规范

- 路由函数只做入参校验和调用服务层。
- 训练任务必须异步化，不允许 HTTP 请求阻塞等待长训练。
- 模型版本必须记录指标、来源数据集、产物地址和兼容运行时。
- 推理接口要返回 request_id、latency_ms、detections。
- GPU 状态、训练日志、失败原因必须可查询。

## 测试要求

- 前端每次提交至少跑 `npm run build`。
- 后端每次提交至少跑 `mvn test`。
- Python 服务每次提交至少跑语法检查和基础接口测试。
- 核心流程必须逐步补端到端测试：设备在线 -> 巡检任务 -> 告警 -> 标注 -> 训练 -> 发布。

## Git 要求

- 主分支：`main`。
- 功能分支：`feature/<name>`。
- 修复分支：`fix/<name>`。
- 提交信息建议：`feat:`、`fix:`、`docs:`、`chore:`、`test:`。
- 不提交 `node_modules`、`dist`、`.env`、日志、缓存、真实设计源文件。
