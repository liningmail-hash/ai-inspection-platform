# 测试计划

本文档对应任务 `QA-P0-001：三线验收清单`，用于第一版质量验收、接口冒烟和缺陷记录。测试工程师仅维护测试文档，不直接修改业务代码。

## 1. 测试目标

- 验证厂区摄像头、无人机、车载监控三条接入链路的端到端闭环。
- 覆盖登录、设备接入、视频播放、AI 识别、告警处置、审计和报告关键路径。
- 为新增 API 建立可重复执行的冒烟测试表。
- 统一 P0/P1/P2 缺陷分级和缺陷记录模板。

## 2. 测试范围

### 2.1 本轮覆盖

- 前端工作台：驾驶舱、视频墙、AI 巡检、告警中心、无人机巡检、设备管理、报表统计、系统配置。
- 中心后端 API：认证、设备、集成、视频、AI 任务、告警、无人机、车辆、审计、报告。
- AI 服务：健康检查、单次推理、数据集、训练任务、模型列表。
- 边缘模拟器：健康检查、节点状态、设备状态、推理任务、事件上报。
- Mock 兜底模式：后端不可用时前端仍可展示三线数据。

### 2.2 本轮不覆盖

- 真实厂商 SDK 的生产级连通性和性能验收。
- 真实 RTSP/WebRTC 低延迟播放质量验收。
- 真实无人机远程起降、航线修改、云台控制。
- 72 小时稳定性压测和 100 路视频规模压测。
- 真实客户现场网络、安全策略和设备账号验收。

## 5. 三线测试场景

### 5.1 厂区摄像头链路

| 编号 | 场景 | 前置条件 | 执行步骤 | 预期结果 |
|---|---|---|---|---|
| CAM-001 | 摄像头集成配置展示 | 已有摄像头或 Mock 集成数据 | 登录后进入设备管理，查看 SDK 集成卡和摄像头通道 | 能看到厂区摄像头来源、厂商、在线状态、通道数和同步入口 |
| CAM-002 | 摄像头通道视频会话 | 存在 `sourceType=camera` 通道 | 在视频墙选择厂区摄像头通道并创建播放会话 | 返回播放地址，页面显示会话状态；失败时有可读错误 |
| CAM-003 | 摄像头抓图与证据 | 摄像头通道可用 | 对通道执行抓图 | 返回证据对象地址，告警或素材可引用该地址 |
| CAM-004 | 摄像头 AI 任务创建 | 存在可用算法和模型版本 | 对摄像头通道创建 AI 任务 | 返回任务 ID 和状态，任务进入 `queued` 或 `running` |
| CAM-005 | 摄像头 AI 告警闭环 | 边缘可上报 AI 事件 | 上报烟火/人员闯入事件，进入告警中心处理 | 告警含来源、证据、级别；状态可从 `new` 流转到 `closed` |
| CAM-006 | 摄像头审计追踪 | 已执行同步、开流、抓图或告警处理 | 查看系统审计日志 | 审计日志记录操作者、动作、对象和时间 |

端到端验收步骤：

1. 使用 `admin / demo123` 登录。
2. 进入设备管理，确认摄像头集成和通道可见。
3. 在视频墙创建摄像头播放会话。
4. 对同一通道执行抓图和 AI 任务创建。
5. 通过边缘事件或 Mock 数据生成告警。
6. 在告警中心完成确认、派单或关闭。
7. 在报表或审计日志中核对事件、证据和处理记录。

### 5.2 无人机链路

| 编号 | 场景 | 前置条件 | 执行步骤 | 预期结果 |
|---|---|---|---|---|
| DRONE-001 | 机场和无人机状态展示 | 已有机场、无人机、遥测数据 | 进入无人机巡检页 | 能看到机场、无人机、电量、位置、在线状态和告警摘要 |
| DRONE-002 | 航线列表展示 | 已有固定航线 | 查看航线任务区域 | 航线名称、里程、航点数、风险等级和适用算法可见 |
| DRONE-003 | 飞行任务启动 | 存在可启动的飞行任务 | 调用或点击启动任务 | 返回成功或明确失败原因，任务状态更新 |
| DRONE-004 | 无人机视频通道 AI 分析 | 存在 `sourceType=drone` 视频通道 | 创建无人机视频会话并创建 AI 任务 | 任务绑定无人机通道，状态可追踪 |
| DRONE-005 | 无人机事件告警 | 边缘或后端有无人机巡检事件 | 触发隐患事件并进入告警中心 | 告警来源标识为无人机，含航线/任务/证据信息 |
| DRONE-006 | 飞行任务停止和审计 | 存在运行中任务 | 停止飞行任务，查看审计日志 | 停止动作有结果反馈，审计可追踪 |

端到端验收步骤：

1. 登录并进入无人机巡检页。
2. 核对机场、无人机、遥测和航线数据。
3. 启动一条固定航线飞行任务。
4. 打开无人机视频通道并创建 AI 任务。
5. 模拟或等待巡检事件进入告警中心。
6. 处理告警并停止飞行任务。
7. 核对审计日志、告警证据和报表记录。

### 5.3 车载监控链路

| 编号 | 场景 | 前置条件 | 执行步骤 | 预期结果 |
|---|---|---|---|---|
| VEH-001 | 车载终端列表展示 | 已有车辆和 JT/T1078 集成数据 | 进入设备管理或视频墙车载分组 | 能看到车牌/终端号、在线状态、通道数量和最近定位 |
| VEH-002 | 车辆视频通道播放 | 存在 `sourceType=vehicle` 通道 | 创建车载视频播放会话 | 返回播放地址，页面按车载分组展示 |
| VEH-003 | 车辆轨迹查询 | 车辆存在轨迹点 | 查询车辆轨迹 | 返回轨迹点列表，时间、经纬度、速度字段完整 |
| VEH-004 | 车载 AI 任务创建 | 车辆通道和算法可用 | 对车载通道创建 AI 任务 | 任务绑定车辆通道，状态可追踪 |
| VEH-005 | 车辆事件告警 | 边缘可上报车辆事件 | 上报车辆视频 AI 事件 | 告警来源为车载监控，证据和车辆信息可见 |
| VEH-006 | 车载集成同步失败 | JT/T1078 账号或网关不可用 | 执行同步或连通性测试 | 返回明确失败原因，不吞异常，不影响其他来源展示 |

端到端验收步骤：

1. 登录并进入设备管理，确认车载终端可见。
2. 查询车辆通道并创建播放会话。
3. 查询同一车辆轨迹点。
4. 对车载通道创建 AI 分析任务。
5. 模拟车辆事件并在告警中心处理。
6. 查看报表和审计日志是否保留车辆、通道、证据、处置记录。

## 6. 通用验收场景

| 编号 | 场景 | 验收点 |
|---|---|---|
| GEN-001 | 登录成功 | 正确账号返回 token，前端进入工作台 |
| GEN-002 | 登录失败 | 错误账号或密码返回明确错误，不进入系统 |
| GEN-003 | 权限与当前用户 | `/auth/me` 返回用户、角色和权限摘要 |
| GEN-004 | Mock 兜底 | 后端不可用时前端仍展示三线演示数据 |
| GEN-005 | 告警状态流转 | `new`、`processing`、`false_positive`、`dispatched`、`closed` 合法流转 |
| GEN-006 | 模型发布 | 模型状态可发布到 `canary` 或 `production`，并记录审计 |
| GEN-007 | 巡检任务创建 | 即时、定时、循环任务参数校验正确 |
| GEN-008 | 报告和素材 | 素材、报告列表可查看，证据 URL 不为空 |
| GEN-009 | 错误反馈 | 接口失败时前端显示可理解的错误 |
| GEN-010 | 页面稳定性 | 主要页面无文字重叠、按钮挤压、表格明显溢出 |

## 7. 缺陷分级

| 等级 | 定义 | 示例 |
|---|---|---|
| P0 阻断 | 核心链路不可用、数据丢失、安全风险、服务无法启动或无法登录 | 登录完全失败；告警无法创建或处理；后端启动失败；接口吞异常导致前端白屏 |
| P1 严重 | 主要功能异常但有绕行方案，影响演示或联调效率 | 某一来源视频会话失败；AI 任务状态不更新；审计缺关键动作 |
| P2 一般 | 局部体验、文案、边界提示或非核心数据问题 | 表格轻微溢出；错误提示不够清晰；统计数字刷新不及时 |
