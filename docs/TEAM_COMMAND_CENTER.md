# 团队协同与总指挥执行手册

本文档是本项目的团队执行总入口。所有开发、测试、文档、部署和设备接入工作，都按本文档领取任务、执行、回报和等待下一步安排。

项目目标：建设一个用于生产安全园区的 AI 巡检平台，覆盖厂区摄像头、无人机、车载监控三类入口，实现全访问监控、AI 识别、告警处置、证据留存、巡检报告和模型迭代闭环。

## 1. 指挥原则

- 总指挥只设一人：由 Codex 担任。
- 所有人只执行总指挥下发的任务单，不互相临时改方向。
- 每个任务单必须明确负责人、目标、文件范围、禁止改动范围、验收标准和回报格式。
- 任何接口字段、数据库结构、跨模块行为变更，必须先回报总指挥，由总指挥重新下发指令。
- 不允许两个人同时修改同一批文件；如果确实需要接力，必须先由总指挥安排交接。
- 任务完成不等于可以继续自由发挥；完成后必须回报，等待下一条任务。

## 2. 角色分工

### 总指挥：Codex

职责：

- 拆分产品目标和技术路线。
- 定义任务边界、接口契约、文件归属和优先级。
- 接收所有人的回报，判断是否通过、返工或进入下一步。
- 处理冲突、阻塞、接口变更和优先级调整。
- 维护路线图、接口文档、验收标准和执行手册。

回报给团队的内容：

- 下一轮任务单。
- 是否通过验收。
- 是否需要返工。
- 新的接口/字段/文件边界。

### 程序员 A：后端与数据库负责人

固定负责范围：

- `backend/`
- `database/`
- `deploy/docker-compose.yml` 中后端、数据库、MQTT、视频服务、对象存储相关内容
- `docs/API_CONTRACTS.md` 中后端接口变更草案

主要职责：

- 后端 REST API、Service、Repository、Domain。
- PostgreSQL 表、迁移脚本、初始化数据。
- 厂家 SDK 适配层：海康、大华、大疆机场、JT/T1078 车载网关。
- JWT、RBAC、审计日志。
- 视频会话、AI 任务、边缘事件、车辆轨迹、无人机任务持久化。

禁止事项：

- 不直接改 `frontend/` 页面。
- 不改变前端字段命名，除非总指挥批准。
- 不把厂家 SDK 调用写进 Controller 或 Service，必须放在 `adapter/`。
- 不把真实密码、Token、RTSP 地址、客户 IP 写入代码或文档。

### 程序员 B：前端与业务工作台负责人

固定负责范围：

- `frontend/`
- `docs/UI_GUIDELINES.md` 中 UI 约束草案
- 需要接口变更时，只能提交“接口需求回报”，不能自行改后端

主要职责：

- 视频墙、设备管理、无人机巡检、车载监控、告警中心、报表地图。
- `frontend/src/services/api.ts` 中统一接口调用。
- API 数据到页面展示模型的适配。
- 页面交互、状态反馈、错误提示、加载态和 Mock 兜底。

禁止事项：

- 不直接拼接后端未定义接口。
- 不在页面组件里写业务真规则。
- 不绕过 `frontend/src/services/api.ts` 请求后端。
- 不修改 `backend/`、`database/`、`deploy/`。

### 测试工程师：质量与验收负责人

固定负责范围：

- `docs/TEST_PLAN.md`
- `docs/ACCEPTANCE_CHECKLIST.md`
- 测试记录、缺陷记录、接口冒烟清单

主要职责：

- 建立三线验收场景：厂区摄像头、无人机、车载监控。
- 验证端到端链路：设备接入 -> 视频播放 -> AI 识别 -> 告警 -> 处置 -> 审计 -> 报告。
- 每轮执行构建、接口、页面和稳定性检查。
- 输出可复现缺陷，不直接改变需求优先级。

禁止事项：

- 不直接改业务代码，除非总指挥明确安排。
- 不要求程序员私下改方向。
- 不用口头描述替代缺陷复现步骤。

### 机动人员：专项支援负责人

固定原则：

- 每次只领取一个专项。
- 文件范围由总指挥临时指定。
- 不进入程序员 A/B 正在修改的文件。

可承担专项：

- 设备资料整理：海康、大华、大疆、JT/T1078 SDK 接入参数。
- 部署资料：Ubuntu、Docker Compose、端口、环境变量。
- 演示数据：SQL 种子、Mock 数据、脱敏样例。
- 文档整理：安装手册、接口说明、现场实施指南。
- AI 资料：模型包规范、数据集目录、标注流程。

## 5. 冲突处理规则

必须暂停并回报总指挥的情况：

- 需要修改别人负责的目录或文件。
- 发现接口字段不够用。
- 发现数据库表结构需要变更。
- 发现当前任务会影响其他人的验收。
- 真实设备、SDK、密钥、网络环境不可用。
- 测试发现严重问题，需要改变实现方案。

禁止自行处理的情况：

- 前端人员私自改后端字段。
- 后端人员私自改页面结构。
- 测试人员直接要求某个程序员改需求。
- 机动人员临时帮忙改核心代码。
- 多人同时修改同一个文件解决冲突。

冲突回报格式：

```text
冲突类型：文件冲突 / 接口冲突 / 数据库冲突 / 需求冲突 / 环境阻塞
涉及人员：
涉及文件：
当前任务编号：
冲突说明：
我建议的处理方式：
是否已暂停：是
```

## 6. 第一轮任务安排

### BE-P0-001：真实 SDK 适配框架

负责人：程序员 A

任务目标：

- 在现有 `CameraSdkAdapter`、`VehicleVideoAdapter` 基础上，形成可插拔厂家 SDK 适配框架。
- 首批预留海康/大华摄像头、大疆机场、JT/T1078 车载网关实现类。

允许修改：

- `backend/src/main/java/com/aiplatform/inspection/adapter/`
- `backend/src/main/java/com/aiplatform/inspection/service/`
- `backend/src/main/resources/application.yml`
- `docs/API_CONTRACTS.md`

禁止修改：

- `frontend/`
- `ai-service/`
- `edge-simulator/`

执行步骤：

1. 保留 Mock 适配器作为无设备演示模式。
2. 新增厂家适配类骨架，例如 `HikvisionCameraSdkAdapter`、`DahuaCameraSdkAdapter`、`DjiDockAdapter`、`Jt1078VehicleAdapter`。
3. 用配置项选择启用哪个厂家适配器。
4. 所有真实 SDK 返回值转换成平台统一对象：`IntegrationConfig`、`VideoChannel`、`VideoSession`、`CommandResult`。
5. 失败时返回明确错误，不允许吞异常。

验收标准：

- Mock 模式保持可用。
- 切换配置后能走指定厂家适配器。
- Controller 接口不变。
- 审计日志记录连通性测试、同步、开流、抓图、录像。

回报给总指挥：

- 适配器类名。
- 配置项名称。
- 是否需要真实 SDK jar、动态库或账号。
- 自测接口结果。

### FE-P0-001：三线工作台打磨

负责人：程序员 B

任务目标：

- 把视频墙、设备管理、告警中心、无人机页打磨成可演示的三线工作台。

允许修改：

- `frontend/src/views/`
- `frontend/src/services/api.ts`
- `frontend/src/data/mock.ts`
- `frontend/src/style.css`

禁止修改：

- `backend/`
- `database/`
- `ai-service/`
- `edge-simulator/`

执行步骤：

1. 视频墙按厂区、无人机、车载分组展示。
2. 设备管理展示 SDK 集成状态、同步按钮、通道数量、车载终端。
3. 告警中心展示证据 URL、来源类型、处理按钮反馈。
4. 无人机页展示机场、无人机、航线、飞行任务、遥测。
5. 所有接口调用都放在 `frontend/src/services/api.ts`。

验收标准：

- `npm run build` 通过。
- 后端不可用时 Mock 兜底可展示。
- 页面没有文字重叠、按钮挤压、表格明显溢出。
- 三类来源都能在界面看到。

回报给总指挥：

- 完成页面。
- 是否需要新增后端字段。
- 构建结果。
- UI 遗留问题。

### QA-P0-001：三线验收清单

负责人：测试工程师

任务目标：

- 建立第一版验收清单和接口冒烟测试表。

允许修改：

- `docs/TEST_PLAN.md`
- `docs/ACCEPTANCE_CHECKLIST.md`

禁止修改：

- 业务代码目录。

执行步骤：

1. 按厂区摄像头、无人机、车载监控三条链路分别列测试场景。
2. 为每条链路写端到端验收步骤。
3. 为新增 API 写冒烟测试表。
4. 定义 P0 阻断级、P1 严重级、P2 一般级缺陷标准。

验收标准：

- 每条链路至少 5 个测试场景。
- 覆盖登录、视频、AI、告警、审计、报告。
- 缺陷模板可直接复制使用。

回报给总指挥：

- 测试文档路径。
- 覆盖场景数量。
- 当前缺失测试条件。
- 需要程序员配合的数据或接口。

### SP-P0-001：真实设备资料整理

负责人：机动人员

任务目标：

- 整理首批真实设备接入需要的信息，供程序员 A 实现真实适配器。

允许修改：

- `docs/DEVICE_SDK_REQUIREMENTS.md`

禁止修改：

- 所有代码目录。

执行步骤：

1. 整理海康/大华摄像头或平台网关需要的接入参数。
2. 整理大疆机场 API/SDK 需要的接入参数。
3. 整理 JT/T1078 车载网关需要的接入参数。
4. 标明哪些参数是现场必须提供，哪些可以默认。

验收标准：

- 每类设备都有参数清单。
- 明确账号、密钥、地址、端口、协议、回调地址、网络要求。
- 不记录真实密钥，只写占位符和获取方式。

回报给总指挥：

- 文档路径。
- 缺少的现场信息。
- 影响开发的 SDK 限制。
- 建议优先接入的厂家。

## 8. 合并顺序

默认合并顺序：

1. 后端接口和数据库。
2. 前端接口适配和页面。
3. AI 服务和边缘模拟器。
4. 测试文档和验收清单。
5. 部署文档和运维文档。

如果前端需要提前开发，必须使用 `frontend/src/data/mock.ts` 中的 Mock 数据，不得自行定义最终接口字段。

## 9. 当前项目优先级

P0：

- 三线统一接入模型。
- 厂家 SDK 适配框架。
- 真实视频开流、抓图、录像。
- AI 任务创建和事件上报。
- 告警证据和处置闭环。
- JWT/RBAC/审计。

P1：

- 真实设备批量同步。
- MQTT 任务下发和断网补传。
- 巡检报告导出。
- 车辆轨迹回放。
- 模型灰度和回滚。

P2：

- Cesium 三维。
- 正射/倾斜摄影。
- GPU 集群。
- Kubernetes。
- 算法插件市场。

## 10. 总指挥继续安排工作的条件

总指挥收到以下信息后，才能继续安排下一步：

- 任务编号。
- 完成状态。
- 修改文件。
- 自测结果。
- 是否有接口/字段/数据库变化。
- 是否需要其他人配合。

如果回报信息不完整，总指挥应先要求补充，不应直接安排下一步。

## 11. 总指挥项目汇报机制

总指挥不仅负责分派任务，也必须向项目负责人同步项目阶段、部署判断、重大风险和下一步安排。

必须汇报的场景：

- 每个 P0 任务完成、部分完成或阻塞。
- 出现 P0 阻断缺陷。
- 项目可以进入部署、试点或验收时。
- 项目不能部署，且原因需要负责人协调时。
- 真实设备、SDK、账号、网络、部署环境发生变化时。
- 完成 Mock 冒烟、内网部署、真实海康单路闭环等里程碑时。

汇报统一写入或引用 [项目状态与部署决策报告](PROJECT_STATUS.md)。

汇报格式：

```text
项目阶段：
本轮完成：
当前可部署状态：
不能部署的原因：
主要风险：
需要你协调：
下一步安排：
预计进入下一阶段时间：
```
