# 系统组件规划

本文档说明 AI 巡检平台从一期骨架到生产版所需的关键组件，以及为什么当前没有一次性全部引入。

## 当前已具备组件

| 组件 | 当前实现 | 作用 |
|---|---|---|
| 前端工作台 | Vue 3 + TypeScript + Vite | 页面、操作台、预览入口 |
| 中心后端 | Spring Boot 3 | 业务 API、查询、动作流转 |
| AI 服务 | FastAPI 骨架 | 推理、训练、模型接口预留 |
| 边缘模拟器 | FastAPI 骨架 | 模拟边缘节点、设备状态、事件 |
| 数据库 | PostgreSQL 初始化脚本 | 业务数据 |
| 对象存储 | MinIO Compose | 图片、视频、模型文件预留 |
| 缓存 | Redis Compose | 缓存和队列预留 |
| 部署 | Docker Compose | 私有化预览部署 |

## 建议新增组件

### 1. 视频服务

候选：

- ZLMediaKit
- SRS
- MediaMTX

用途：

- RTSP 拉流。
- HLS/WebRTC/FLV 播放。
- 截图、录像、视频片段。
- 后续承接 GB28181 或无人机直播流。

建议阶段：P0。

### 2. 国标 GB28181 服务

候选：

- WVP-GB28181-pro
- 自研 SIP 接入服务

用途：

- GB28181 设备注册。
- SIP 信令。
- 国标设备目录。
- 点播、回放、云台控制。

建议阶段：P1，真实摄像头接入前必须明确。

### 3. MQTT 消息服务

候选：

- EMQX
- Mosquitto

用途：

- 边缘节点心跳。
- 任务下发。
- AI 事件上报。
- 断网补传协调。

建议阶段：P1。

### 4. AI 任务队列

候选：

- Celery + Redis
- RQ + Redis
- Dramatiq

用途：

- 训练任务异步执行。
- 推理批处理。
- 模型转换任务。
- 失败重试和任务日志。

建议阶段：P0/P1。

### 5. 标注工具

候选：

- CVAT
- Label Studio
- 自研轻量标注

用途：

- 图片/视频帧标注。
- 标注质检。
- 数据集导出。

建议阶段：P1。若要快速开源可先接 Label Studio，若要产品一致性可自研轻量标注。

### 6. 地图组件

候选：

- MapLibre GL
- 高德地图 JS API
- 百度地图

用途：

- 设备、机场、事件分布。
- 航线绘制。
- 禁飞区、飞行区。

建议阶段：P0/P1。

### 7. 三维组件

候选：

- CesiumJS
- 3D Tiles

用途：

- 实景三维。
- 倾斜摄影模型。
- 三维航线规划。

建议阶段：P2。

### 8. 认证和权限

候选：

- Spring Security + JWT
- OAuth2/OIDC
- Keycloak

用途：

- 登录。
- 用户、角色、组织。
- 菜单权限。
- 数据权限。
- 审计。

建议阶段：P0。

### 9. 监控和日志

候选：

- Prometheus
- Grafana
- Loki
- OpenTelemetry

用途：

- 服务健康。
- GPU 指标。
- 视频通道状态。
- 任务失败原因。
- 日志检索。

建议阶段：P1。

## 为什么不一开始全部引入

这些组件都重要，但过早全部引入会导致：

- 部署门槛过高。
- 没有真实设备时无法验证。
- 服务太多，问题定位困难。
- 开源用户第一步跑不起来。
- 架构边界还没稳定时会造成长期技术债。

因此当前策略是：

1. 保持核心业务闭环可运行。
2. 每次只引入能立即验证价值的组件。
3. 所有外部组件都通过适配层接入。
4. 先 Docker Compose 跑通，再考虑 Kubernetes。

## 下一批推荐引入

优先级从高到低：

1. 认证/RBAC/审计。
2. 视频服务 ZLMediaKit 或 SRS。
3. 地图组件 MapLibre GL。
4. 媒体资产和报告模块。
5. MQTT/EMQX 云边消息。
6. AI 任务队列 Celery/RQ。
