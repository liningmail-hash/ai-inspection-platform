## Device Management CRUD

Device management supports real backend writes in both default InMemory Mock
mode and `postgres` profile. Sensitive credentials are not accepted or stored as
plain text; only `credentialRef` is stored.

Device fields:

| field | description |
|---|---|
| `id` | Device id generated by backend |
| `name` | Device display name |
| `sourceType` | `camera`, `drone`, or `vehicle` |
| `vendor` | `MOCK_VENDOR`, `HIKVISION`, `DAHUA`, `DJI_DOCK`, `JT1078`, etc. |
| `protocol` | SDK/protocol label, such as `RTSP`, `HIK_SDK`, `DJI_SDK`, `JT1078` |
| `endpoint` | SDK gateway, RTSP URL, OpenAPI endpoint, or gateway endpoint |
| `credentialRef` | Secret reference only, never plain credentials |
| `location` | Human-readable device location |
| `edgeNodeId` | Edge node binding, such as `EDGE-01` |
| `status` | `offline`, `online`, `error`, or `deleted` |
| `streamUrl` | Raw stream URL when available |
| `createdAt` | Backend created timestamp |
| `updatedAt` | Backend updated timestamp |

```http
POST /api/devices
Content-Type: application/json
```

{
  "name": "Area A new camera",
  "sourceType": "camera",
  "vendor": "MOCK_VENDOR",
  "protocol": "RTSP",
  "endpoint": "rtsp://example/live/new-camera",
  "credentialRef": "secret://integrations/new-camera",
  "location": "Area A warehouse",

}

## 状态值约定

- 通用状态：`online`、`offline`、`warning`、`error`。
- 告警状态：`new`、`processing`、`false_positive`、`dispatched`、`closed`。
- 任务状态：`queued`、`running`、`success`、`failed`、`cancelled`。
- 模型状态：`candidate`、`canary`、`production`、`archived`。
- 巡检计划状态：`enabled`、`disabled`、`paused`。

## 前端映射规则

- 后端状态值保持英文稳定枚举。
- 前端负责将状态映射为中文展示。
- 页面不得直接依赖后端原始枚举文本做中文判断。
- API 调用集中在 `frontend/src/services/api.ts`。
- Mock 数据集中在 `frontend/src/data/mock.ts`，只作为后端不可用时的兜底。

## Backend SDK adapter selection

The public REST API remains unchanged. Backend vendor SDK calls must stay behind
`backend/src/main/java/com/aiplatform/inspection/adapter/`.

Runtime vendor selection is controlled by:

- `platform.sdk.camera-vendor` / `PLATFORM_CAMERA_VENDOR`, default `MOCK_VENDOR`
- `platform.sdk.drone-vendor` / `PLATFORM_DRONE_VENDOR`, default `MOCK_VENDOR`
- `platform.sdk.vehicle-vendor` / `PLATFORM_VEHICLE_VENDOR`, default `MOCK_VENDOR`

Reserved adapter vendor names:

- `MOCK_VENDOR`
- `HIKVISION`
- `DAHUA`
- `DJI_DOCK`
- `JT1078`

Real vendor adapters currently return explicit unavailable results until the
required SDK jars/native libraries, endpoint credentials, and gateway accounts
are provided. Mock mode remains the no-device demonstration mode.

### HIKVISION camera adapter preparation

Enable the adapter with `PLATFORM_CAMERA_VENDOR=HIKVISION`. Public REST paths do
not change.

HIKVISION configuration placeholders:

- `platform.sdk.hikvision.endpoint` / `HIKVISION_ENDPOINT`: ISUP/OpenAPI/SDK gateway endpoint. `IntegrationConfig.endpoint` takes precedence when present.
- `platform.sdk.hikvision.username` / `HIKVISION_USERNAME`: SDK login account name.
- `platform.sdk.hikvision.credential-ref` / `HIKVISION_CREDENTIAL_REF`: secret reference for password, app secret, or token. `IntegrationConfig.credentialRef` takes precedence when present.
- `platform.sdk.hikvision.sdk-library-path` / `HIKVISION_SDK_LIBRARY_PATH`: local path containing HCNetSDK jar and native libraries.
- `platform.sdk.hikvision.stream-mode` / `HIKVISION_STREAM_MODE`: preferred stream mode, default `rtsp`.
- `platform.sdk.hikvision.callback-base-url` / `HIKVISION_CALLBACK_BASE_URL`: backend callback base URL for alarms, snapshots, recordings, or stream callbacks.

HIKVISION channel sync maps vendor channel data into `VideoChannel` as follows:

- vendor device id -> `sourceId`
- vendor channel id -> `id`
- vendor channel name -> `name`
- main/sub stream type -> `streamName`
- vendor protocol or SDK type -> `protocol`
- raw stream address -> `rawUrl`
- playable platform URL -> `playUrl`
- vendor online flag -> `online` and `status`
- edge node binding -> `edgeNode`
- device location -> `latitude` and `longitude`

When HCNetSDK jars/native libraries or required configuration are missing,
HIKVISION operations return explicit unavailable messages. Vehicle channels do
not use camera snapshot or record operations; unsupported vehicle operations
return a failed command result from the vehicle adapter boundary.
