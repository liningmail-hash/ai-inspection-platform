# 路线图

本路线图只定义阶段、上线口径和交付内容，不固定日历日期。所有实际时间节点以项目负责人最终确认为准。

## 2026-06-16 实施更新

本轮已按“三线并行、厂家 SDK 优先”落地 P0 骨架：

- 厂区摄像头、无人机、车载监控统一为 `sourceType=camera|drone|vehicle`，统一挂视频通道、AI任务、地图坐标和证据资产。
- 后端新增集成、视频会话、AI任务、边缘心跳/事件、无人机、航线、车载终端和轨迹接口。
- 适配器层新增 `CameraSdkAdapter`、`VehicleVideoAdapter` 和 Mock 厂家实现，后续海康/大华/大疆/JT1078 真实 SDK 只替换适配层。
- Compose 增加 SRS 视频服务、EMQX MQTT 和 MinIO 桶初始化。
- 前端视频墙、设备管理、无人机、驾驶舱、报表页已能展示厂区/无人机/车载三线数据。

下一步建议按优先级推进：

1. 接入第一套真实厂家 SDK，优先海康或大疆机场，验证连通、通道同步、开流、抓图。
2. 将 SRS 播放地址接入真实转流，完成一路真实厂区视频、一路无人机视频、一路车载视频预览。
3. 将 AI 服务替换为 ONNX Runtime/TensorRT 推理实现，保留当前请求/响应契约。
4. 将边缘模拟器升级为真实边缘 Agent，补 MQTT 任务下发、断网缓存和事件补传。

## Phase 1：可运行工程骨架

- 前端工作台：驾驶舱、视频监控、AI巡检、告警、标注训练、模型、无人机、设备、报表、系统配置。
- 中心后端：核心 REST API、设备适配器、无人机适配器、演示数据服务。
- AI 服务：推理、数据集、训练任务、模型版本接口。
- 边缘模拟器：节点状态、设备状态、推理任务、事件上报接口。
- 部署基线：PostgreSQL、Redis、MinIO、后端、AI服务、边缘模拟、前端 Compose。
- 开源材料：README、安装部署、开发规范、API契约、组件规划、功能比对。

## Phase 2：一期基础闭环

- 前端从 Mock 数据切换到后端 API，保留 Mock 兜底能力。
- 后端接入 PostgreSQL 持久化实现。
- 登录、RBAC、用户、角色、组织、站点、审计日志。
- 告警状态流转、模型发布、训练任务创建形成端到端链路。
- 任务创建、媒体素材、巡检报告、算法参数管理。
- 二维地图事件分布。

## Phase 3：视频与真实协议接入

- 引入视频服务，完成 RTSP、HLS、WebRTC 预览链路。
- 海康、大华、ONVIF、GB/T 28181、JT/T 1078 适配验证。
- 截图、证据短视频、实时识别叠框、AI 事件入库。
- MQTT 云边消息通道。
- 首个无人机机场厂商适配。
- 航线导入导出和飞行记录回放。

## Phase 4：无人机巡检增强

- 机场状态、电池、天气、设备健康、远程调试基础能力。
- 固定航线巡检、即时任务、定时任务、循环任务。
- 无人机视频 AI 分析、任务报告、媒体自动上传。
- 机库控制、载荷控制、指点飞行进入受控验证。

## Phase 5：试点验收

- 50-150 路设备接入验证。
- 10-20 路并发 AI 推理验证。
- 72 小时稳定性压测。
- 告警闭环、样本标注、训练发布、无人机巡检报告验收。
- Ubuntu 私有化部署和运维手册验收。

## Phase 6：二期/三期扩展

- API 网关、消息队列、统一监控、日志检索。
- Cesium 三维地图、实景三维航线规划、限飞区、飞行区。
- 二维正射图、三维模型、模型对比、照片建模。
- Kubernetes、GPU 集群训练、多站点集团化扩展。
- 专业场景算法和厂商插件生态。
